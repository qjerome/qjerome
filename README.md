## My Usage of AI

I believe transparency about AI usage is important as it provides context for evaluating work quality. Here's my approach:

- **Documentation**: As a non-native English speaker, I use AI to create clear, consistent documentation while ensuring technical accuracy. I always use it to document already trusted/reviewed content.

- **Simple Algorithms**: For repetitive, straightforward code that offers no technical challenge, AI can save time but I always review the output thoroughly.

- **Writing Review**: AI helps proofread my English writing to enhance quality, but I never rely on fully AI-generated content. Rather, I write the full content and then use AI to proofread. I believe this process reduces hallucination effect and thus minimizes my review time.

- **Search Engine**: AI efficiently finds information (though not always accurately).

**Key Principle**: I maintain near-zero trust in AI output. Every generated code undergoes functional and performance review. For writing activities (media/blog posts), I author the core content myself and use AI only for refinement - except for code documentation where the code itself is the primary content.
